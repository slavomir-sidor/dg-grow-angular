• 1. The application should be self-contained, without requiring access to any backend.
  • 2. The only external data provider allowed is the provided JSON file.
  • 3. The application must have a login page with a mock validation where
    • a. It returns an error if the username and password are not valid.
    • b. It directs the user to “welcome” page if the login is successful.
  • 4. The Welcome page must contain links to the following pages:
    • a. New Product
    • b. Sales
    Please note that links should not be implemented via direct HTML hardcode but with routing. The links may be presented in a toolbar.
  • 5. The “New Product” page must have a form with the following information:
    Form field Type Dimension Is required
    Product name String 50 Yes
    Product ID Integer 13 Yes
    Product manager String 30 No
    Sales start date Date 10 Yes
    The form should validate for correct content and mandatory fields. Submit and clear buttons must be available.
  • 6. The “Sales” page should display a data grid with the following header:
    Product ID Product Name Sales Total sales
    2019Q1 2019Q2 2019Q3 2019Q4
    • a. Information to populate the grid should be retrieved from the provided JSON file.
    • b. Column names should not be hardcoded in the template but retrieved from the provided JSON file.
    • c. Total sales value is calculated from the information retrieved from the JSON file.
    • d. The data should be sortable per column.
    • e. The table should be searchable (simple global search).
  • 7. As an improvement, the data entered in the in the “New product” form could be added to the data table (in runtime, not on a permanent basis), and the grid should provide a way to enter data for the new product, as well as edit the data already displayed. There is no need to save changes to the information.
